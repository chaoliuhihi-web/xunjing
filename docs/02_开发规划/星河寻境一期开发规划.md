# 星河寻境一期上线开发规划

更新时间：2026-06-21

## 1. 规划结论

一期不从零搭后台，不另造 AI 管理、知识库、图片库和资料采集审核底座。

一期采用“独立 Yudao 后台 + UniApp 真实前端”的路线：直接以 `/Users/bruce/Developer/work/XingheAI2026V2` 中已存在的 Yudao、AI 管理、知识库服务、采集网关和本地模型配置能力为迁移参考，在星河寻境项目中复制一份独立 Yudao，收敛成面向 AI 文旅的内容后台、AI 编排、知识资产库、图影中华图片库、资料导入与采集审核和二维码入口网络。

一期核心目标不是做截图演示，也不是只做假数据 UI，而是在 6-8 周内先打透“图秀中华公益行动·新疆首站”的最小闭环。`assets/references/APP/参考图` 是视觉验收标准，不是最终实现方式；`assets/references/APP/kashgar-mini-program/static/demo/kashgar` 中的整图热区模式只能作为视觉对照和临时评审工具，不能作为上线交付。

一期 P0 不追求完整文旅平台，而是先跑通：

```text
一本书 / 一张地图 / 一个地球仪 -> 扫码 -> 权威讲解 -> AI 问答有来源 -> 使用数据回传 -> 公益报告生成
```

P0 必须完成：

1. 独立 Yudao 跑通。
2. `yudao-module-xunjing` 基础业务模块。
3. 权威内容知识库最小闭环。
4. 二维码绑定系统。
5. 图书扫码伴读。
6. 地图 / 景区扫码讲解。
7. RAG 问答必须带来源。
8. 基础素材库：来源、版权、审核、可用开关、调用日志。
9. 学生扫码学习入口。
10. 项目管理后台。
11. 基础数据看板。
12. 公益报告样例。

P1 试点增强：

1. AI 旅行记录。
2. 游记生成。
3. 研学报告生成。
4. 简版 PDF 纪念页。
5. 地球仪语音样机联动。
6. 活动专题页。
7. 素材 AI 标注。
8. 简单文旅路线和打卡任务。

P2 平台化能力：

1. 完整合规爬虫体系。
2. 图片向量相似搜索。
3. 视频关键帧与视频摘要。
4. 多地区复制模板。
5. 多租户运营。
6. 内容包市场。
7. 文旅报告自动生成。
8. 复杂设备管理、APP、C 端付费和电商。

上线口径：

- 前端必须是可维护的 UniApp 页面、组件、状态和接口调用，不允许默认展示整张参考图。
- P0 前端只要求启动/首页、扫码解析、地图/景区讲解、AI 问答、图书伴读 5 个页面闭环；旅行日程、游记生成、视频详情、内容流、我的作品后置到 P1。
- 后台必须用 Yudao 做真实内容管理，不允许长期依赖 mock JSON。
- Mock 只允许用于前端第一轮联调，且必须有同名 Yudao 接口替换计划。
- 所有 AI Key、Token、模型配置进入本地未提交环境变量或 Yudao AI 管理，不写入源码。

## 6. 核心数据策略

### 6.1 不直接改烂 Yudao AI 表

Yudao 原生 AI 表负责“模型、Key、AI 角色、知识库基础能力”。星河寻境业务表负责“地区、景区、图书、二维码、旅行记录、文旅影像资产和作品”。

### 6.2 知识库分层

知识库采用四层：

1. 原始资料层：官方文档、图书章节、网页采集结果、人工录入资料。
2. 结构化知识层：标题、地区、主题、景区、图书、来源、版权、审核状态。
3. AI 表达层：儿童版、游客版、伴读版、游记素材版、宣传版。
4. 应用调用层：景区讲解、问问 AI、扫码伴读、旅行记录生成、运营内容生成。

### 6.3 影像库分层

影像库采用三层：

1. 原始素材库：原图、视频、版权、来源、拍摄者、授权范围。
2. AI 标注与解读库：P1/P2，标签、描述、关键帧、视频摘要、拍照建议、人工说明。
3. 成品素材库：P1/P2，可用于游记、伴读、小红书、朋友圈、宣传图、报告和 PDF 纪念页的精选素材。

P0 只要求基础素材库：上传、来源、版权、审核、可用开关、调用日志。

## 9. 验收门禁

### 9.1 Git 门禁

- 目标仓库必须有 GitHub 和 Gitee 两个远端。
- 每次正式推送必须同时推 GitHub 和 Gitee。
- 迁移 XingheAI2026V2 代码时必须记录源路径和源 commit。
- 不允许提交 `.env`、`.runtime`、真实密钥、Office 锁文件、大型临时生成文件。

### 9.2 后台门禁

- Yudao 能登录。
- 菜单权限可控。
- 文件上传可用。
- AI 模型配置可用。
- 星河寻境业务模块不破坏 Yudao 原生模块。

### 9.3 AI 门禁

- 模型调用失败时有明确错误信息。
- RAG 输出必须带来源。
- 固定 AI 评测集必须通过，覆盖新疆、版图、民族、宗教、地图边界、不知道的问题处理。
- 必须有单用户、单二维码、单学校、单项目的调用上限和预算上限。
- 高频问答缓存和地球仪/地图常见语音预生成必须可配置。
- 图片理解和语音识别属于 P1，失败时不阻塞用户保存原始记录。
- 每次 AI 生成记录任务类型、模型、输入摘要、输出摘要、token、成本和安全状态。

### 9.4 知识库门禁

- 元数据主存是 MySQL。
- Qdrant 只做向量索引。
- 知识库每条内容有来源、审核状态和可调用场景。
- 未审核内容不能进入正式问答。

### 9.5 图片库门禁

- 每个素材有版权归属和授权范围。
- 未标记 `can_ai_use` 的素材不能进入 AI 生成。
- 未标记 `can_public` 的素材不能公开展示。
- 图片和视频被调用时记录调用来源。

### 9.6 资料导入与采集审核门禁

- 只导入授权或公开可用资料。
- 所有导入结果先进入待审核。
- 每次导入保留来源 URL 或来源说明、时间、版本、审计记录。
- 不绕登录、不采集隐私、不绕版权。
- P0 不做通用站点爬虫、定时爬虫、公众号采集和大规模网页发现。

## 10. 近期最小可执行任务

如果今天开始开发，建议按下面顺序执行：

1. 建立 GitHub 远端，完成 GitHub/Gitee 双远端配置。
2. 创建初始 commit，提交现有规划、资产索引和本开发规划。
3. 冻结 XingheAI2026V2 迁移源 commit。

6. 在 `yudao-module-xunjing` 内实现项目、学校、资源包、图书、地图、地球仪点位、二维码。
