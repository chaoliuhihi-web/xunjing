# 星河寻境一期上线开发规划

更新时间：2026-06-21

## 1. 规划结论

一期不从零搭后台，不另造 AI 管理、知识库、图片库和爬虫底座。

一期采用“独立 Yudao 后台 + UniApp 真实前端”的路线：直接以 `/Users/bruce/Developer/work/XingheAI2026V2` 中已存在的 Yudao、AI 管理、知识库服务、爬虫网关和本地模型配置能力为迁移参考，在星河寻境项目中复制一份独立 Yudao，收敛成面向 AI 文旅的内容后台、AI 编排、知识资产库、图影中华图片库、合规采集和二维码入口网络。

一期核心目标不是做截图演示，也不是只做假数据 UI，而是在 6-8 周内交付一个可以进入真实试点上线流程的 MVP。`assets/references/APP/参考图` 是视觉验收标准，不是最终实现方式；`assets/references/APP/kashgar-mini-program/static/demo/kashgar` 中的整图热区模式只能作为视觉对照和临时评审工具，不能作为上线交付。

一期必须跑通四条真实业务主线：

1. 景区扫码 AI 旅伴：喀什古城二维码进入景区专属 AI 旅伴，支持欢迎语、景点讲解、官方攻略、打卡地、问问 AI。
2. 扫码伴读：图书或章节二维码进入 AI 伴读空间，支持一分钟讲懂、三个重点、图文趣味讲解、追问、读书笔记、知识卡片。
3. AI 旅行记录：用户上传照片、语音或文字后，生成旅行摘要、亲子游记、朋友圈、小红书、分享卡、PDF 纪念页。
4. Yudao 后台：后台能真实管理知识库、影像资产、景区内容、AI 角色、图书章节、二维码、用户作品、基础数据、AI 成本。

上线口径：

- 前端必须是可维护的 UniApp 页面、组件、状态和接口调用，不允许默认展示整张参考图。
- 参考图中的布局、文字、层级、底部导航、喀什元素、AI 旅伴形象、地图/指南/游记页面都要组件化还原。
- 后台必须用 Yudao 做真实内容管理，不允许长期依赖 mock JSON。
- Mock 只允许用于前端第一轮联调，且必须有同名 Yudao 接口替换计划。
- 所有 AI Key、Token、模型配置进入本地未提交环境变量或 Yudao AI 管理，不写入源码。

## 6. 核心数据策略

### 6.1 不直接改烂 Yudao AI 表

Yudao 原生 AI 表负责“模型、Key、AI 角色、知识库基础能力”。星河寻境业务表负责“地区、景区、图书、二维码、旅行记录、文旅影像资产和作品”。

建议新增星河寻境业务前缀，例如：

```text
xj_region
xj_scenic_area
xj_scenic_spot
xj_official_guide
xj_checkin_spot
xj_book
xj_book_chapter
xj_media_asset
xj_qrcode
xj_trip
xj_trip_record
xj_user_work
xj_ai_generation_log
```

业务表通过以下字段挂接 Yudao AI：

```text
ai_knowledge_id
ai_model_id
ai_chat_role_id
knowledge_base_id
media_library_id
```

### 6.2 知识库分层

知识库采用四层：

1. 原始资料层：官方文档、图书章节、网页采集结果、人工录入资料。
2. 结构化知识层：标题、地区、主题、景区、图书、来源、版权、审核状态。
3. AI 表达层：儿童版、游客版、伴读版、游记素材版、宣传版。
4. 应用调用层：景区讲解、问问 AI、扫码伴读、旅行记录生成、运营内容生成。

### 6.3 影像库分层

影像库采用三层：

1. 原始素材库：原图、视频、版权、来源、拍摄者、授权范围。
2. AI 标注与解读库：标签、描述、关键帧、视频摘要、拍照建议、人工说明。
3. 成品素材库：可用于游记、伴读、小红书、朋友圈、宣传图、报告和 PDF 纪念页的精选素材。

## 9. 验收门禁

### 9.1 Git 门禁

- 目标仓库必须有 GitHub 和 Gitee 两个远端。
- 每次正式推送必须同时推 GitHub 和 Gitee。
- 迁移 XingheAI2026V2 代码时必须记录源路径和源 commit。
- 不允许提交 `.env`、`.runtime`、真实密钥、Office 锁文件、大型临时生成文件。

### 9.2 后台门禁

- Yudao 能登录。
- 菜单权限可控。
- 文件上传可用。
- AI 模型配置可用。
- 星河寻境业务模块不破坏 Yudao 原生模块。

### 9.3 AI 门禁

- 模型调用失败时有明确错误信息。
- RAG 输出必须带来源。
- 图片理解和语音识别失败时不阻塞用户保存原始记录。
- 每次 AI 生成记录任务类型、模型、输入摘要、输出摘要、token、成本和安全状态。

### 9.4 知识库门禁

- 元数据主存是 MySQL。
- Qdrant 只做向量索引。
- 知识库每条内容有来源、审核状态和可调用场景。
- 未审核内容不能进入正式问答。

### 9.5 图片库门禁

- 每个素材有版权归属和授权范围。
- 未标记 `can_ai_use` 的素材不能进入 AI 生成。
- 未标记 `can_public` 的素材不能公开展示。
- 图片和视频被调用时记录调用来源。

### 9.6 爬虫门禁

- 只采集授权或公开可用资料。
- 所有爬虫结果先进入待审核。
- 每次采集保留来源 URL、时间、版本、审计记录。
- 不绕登录、不采集隐私、不绕版权。

## 10. 近期最小可执行任务

如果今天开始开发，建议按下面顺序执行：

1. 建立 GitHub 远端，完成 GitHub/Gitee 双远端配置。

7. 在 `yudao-module-xunjing` 内实现合规采集预览和待审核发布。
