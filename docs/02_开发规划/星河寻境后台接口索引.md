## 2. 小程序 App API

实际网关前缀由 Yudao 配置提供，部署后通常是 `/app-api`。

Yudao 默认启用多租户过滤，小程序调用本节所有接口都必须带请求头：

```http
tenant-id: ${XUNJING_TENANT_ID}
```

本地喀什 P0 seed 的租户编号为 `1`；生产环境使用实际租户编号，不能硬编码本地值。

### 扫码解析

```http
POST /app-api/xunjing/scan/resolve
Content-Type: application/json
```

```json
{
  "sceneCode": "QR-KASHGAR-MAP-001",
  "userTraceId": "anonymous-trace-id"
}
```

返回核心字段：

```json
{
  "code": 0,
  "data": {
    "packageCode": "KASHGAR-MAP-001",
    "sceneCode": "QR-KASHGAR-MAP-001",
    "title": "喀什古城研学地图",
    "resourceType": "MAP",
    "targetPath": "/pages/map/detail?packageCode=KASHGAR-MAP-001&sceneCode=QR-KASHGAR-MAP-001"
  }
}
```

说明：小程序正式扫码入口优先只传 `sceneCode`。后台会查 `xunjing_qrcode`，校验资源包为 `PUBLISHED`，记录 `SCAN` 事件，并增加二维码 `scanCount`。兼容直接传 `packageCode` 的调试方式。

后台二维码 `path` 可以预置渠道参数，例如 `/pages/map/detail?source=offline-card`；扫码解析会自动使用 `&` 追加 `packageCode` 和 `sceneCode`，避免生成无效双问号路径。

### AI 问答和讲解

```http
POST /app-api/xunjing/ai/chat
POST /app-api/xunjing/reading/ask
POST /app-api/xunjing/map/explain
POST /app-api/xunjing/globe/explain
Content-Type: application/json
```

```json
{
  "packageCode": "KASHGAR-MAP-001",
  "question": "给孩子讲讲喀什古城是什么。",
  "sceneCode": "xunjing-rag-chat",
  "qrSceneCode": "QR-KASHGAR-MAP-001",
  "sourceChannel": "mini-program",
  "userTraceId": "anonymous-trace-id"
}
```

返回核心字段：

```json
{
  "code": 0,
  "data": {
    "answer": "根据已审核资料《喀什古城权威讲解稿》：...",
    "safetyStatus": "PASSED",
    "logId": 1,
    "sources": [
      {
        "id": 1,
        "title": "喀什古城权威讲解稿",
        "sourceUrl": "https://example.com/kashgar",
        "score": 0.75
      }
    ]
  }
}
```

说明：公开问答只使用 `reviewStatus=APPROVED` 且 `vectorStatus=INDEXED` 的知识文档，并记录 ASK 事件和 AI 调用成本日志。问答会先执行配额，再检查同日同资源包、同二维码、同用户 trace、同场景、同问题的缓存；命中缓存时返回历史回答和 sources，并记录 `cacheHit=true`、`tokenCount=0` 的 AI 调用日志，不再调用模型。未命中时会召回 sources，再通过本项目内复制的 `yudao-module-ai` 读取默认 Chat 模型和 API Key 生成最终回答；模型未配置、模型异常或返回空内容时，会降级为确定性讲解文本，但仍保留 sources 和日志。`qrSceneCode` 用于把问答归因到具体二维码，前端从扫码解析返回的 `sceneCode` 透传即可。问答会执行 `ai-quota-rules` 配额，支持 `PROJECT`、`SCHOOL`、`PACKAGE`、`QRCODE`、`USER` 五类作用域的日次数与月预算控制；超限时返回 `safetyStatus=BLOCKED`，并记录一条成本为 0 的 BLOCKED 日志。

`/ai/chat` 是通用问答入口，不限制资源类型。`/reading/ask` 只允许 `BOOK` 资源包，`/map/explain` 只允许 `MAP` 资源包，`/globe/explain` 只允许 `GLOBE` 资源包；类型不匹配会拒绝请求，防止小程序页面串资源。

### 公益报告摘要

```http
GET /app-api/xunjing/public-report/summary?packageCode=KASHGAR-MAP-001
```

说明：公开报告摘要以 `packageCode` 所属资源包为入口。资源包绑定学校时，优先返回同项目、同学校最新公益报告；该学校暂无报告时，才回退项目级 `schoolId=null` 的最新报告，避免同一项目下不同学校报告串用。

返回核心字段：

```json
{
  "code": 0,
  "data": {
    "reportId": 1,
    "projectId": 1,
    "schoolId": 1,
    "title": "2026-Q2 喀什公益研学报告",
    "reportPeriod": "2026-Q2",
    "packageCount": 3,
    "reviewedKnowledgeCount": 3,
    "reviewedMediaCount": 3,
    "qrCodeCount": 20,
    "p0Ready": true
  }
}
```

### 回传访问事件

```http
POST /app-api/xunjing/resource/events
Content-Type: application/json
```

```json
{
  "packageCode": "KASHGAR-MAP-001",
  "sceneCode": "QR-KASHGAR-MAP-001",
  "eventType": "VIEW",
  "sourceChannel": "mini-program",
  "userTraceId": "anonymous-trace-id",
  "payloadJson": "{\"page\":\"map-detail\",\"duration\":12}"
}
```

说明：前端可以只传 `sceneCode`，后台会解析有效二维码、校验资源包为 `PUBLISHED`，写入 `packageId`、`schoolId`，并把 `sceneCode`、`qrCodeId` 和原始客户端 payload 放入 `payloadJson`。`eventType=SCAN` 会同步增加二维码 `scanCount`；常规页面访问建议使用 `VIEW`、`SHARE` 或 `MEDIA_USE`。

`eventType` 只允许 `SCAN`、`VIEW`、`ASK`、`SHARE`、`MEDIA_USE`。空值会按 `VIEW` 记录；非法值会被拒绝，避免污染 P0 运营统计。

`MEDIA_USE` 用于小程序曝光或使用图影中华/项目授权素材。此时 `payloadJson` 必须包含 `mediaId`，可选 `usageType`、`placement` 等前端上下文；后台会校验素材属于当前公开资源包，且满足 `APPROVED + AUTHORIZED + canPublic=true`，然后同时写入互动事件表和 `xunjing_media_usage_log`，用于 readiness、公益报告和素材授权留痕统计。

生成公益报告的统计口径：

- `schoolId` 为空：按 `projectId` 统计项目总量。
- `schoolId` 有值：只统计该学校名下资源包，以及这些资源包产生的知识、素材、地图点、地球仪、二维码、互动事件、素材使用、AI 调用和待审核导入项；AI 评测集和配额规则仍按项目配置统计。

创建资源包可选绑定 Yudao AI 知识库：

```json
{
  "projectId": 1,
  "schoolId": 1,
  "packageCode": "KASHGAR-MAP-001",
  "title": "喀什古城研学地图",
  "resourceType": "MAP",
  "versionNo": "v1.0.0",
  "aiKnowledgeId": 7001
}
```

绑定后，App 问答会优先调用 `yudao-module-ai` 的 `AiKnowledgeSegmentService.searchKnowledgeSegment` 召回段落；没有绑定或没有召回结果时，回退到 `xunjing_knowledge_document` 中已审核且已索引的知识文档。最终回答生成复用 `AiModelService.getRequiredDefaultModel(CHAT)` 和 `AiModelService.getChatModel(modelId)`，Prompt 会要求模型“必须基于后台提供的资料来源回答”。

AI 评测运行会真实调用当前 App 问答链路，用于上线前质量门禁：

```json
{
  "evalSetId": 1,
  "packageCode": "KASHGAR-MAP-001",
  "qrSceneCode": "QR-KASHGAR-MAP-001",
  "userTraceId": "staging-eval-001"
}
```

返回结果包含 `totalCount`、`passedCount`、`failedCount`、`passed` 和逐题 `caseId`、`safetyStatus`、`sourceCount`、`logId`、`failureReason`。当评测题 `sourceRequired=true` 但问答没有召回 sources 时，逐题结果会失败并返回 `SOURCE_REQUIRED_BUT_EMPTY`。当评测题 `riskTags` 包含 `unknown_answer`、`unknown-answer`、`insufficient_source` 或 `real_time`，但回答没有明确表达资料不足、无法确认或不能直接回答时，逐题结果会失败并返回 `UNKNOWN_ANSWER_POLICY_NOT_MET`。

AI 配额规则的 `cacheEnabled=true` 会启用 App 问答缓存。缓存不绕过配额：每次请求先过配额检查，再决定是否命中缓存；缓存命中仍会新增一条 AI 调用日志，便于公益报告和成本看板统计真实使用次数。

图影中华素材新增和审核必须带清晰使用开关：

```json
{
  "packageId": 1,
  "title": "喀什古城街巷授权图片",
  "mediaType": "IMAGE",
  "fileUrl": "https://cdn.example.com/tuyingzhonghua/kashgar-street.jpg",
  "sourceProvider": "图影中华",
  "copyrightStatus": "AUTHORIZED",
  "reviewStatus": "APPROVED",
  "canPublic": true,
  "canAiUse": true,
  "canPromotionUse": false
}
```

`canPublic=false` 的素材即使已审核、已授权，也不会出现在 App 公开资源包详情里；`canAiUse=false` 的素材不能进入后续 AI 生成任务；`canPromotionUse=false` 的素材不能进入官网、宣传图、PPT、海报等对外传播物料。

资料导入审核通过后会发布为可引用知识文档：

```json
{
  "id": 1,
  "reviewStatus": "APPROVED",
  "targetType": "KNOWLEDGE",
  "authorityLevel": "OFFICIAL",
  "vectorStatus": "INDEXED",
  "reviewer": "admin",
  "reviewRemark": "来源已确认，可进入 P0 问答"
}
```

审核通过时，后台会从 `xunjing_import_item` 生成一条 `xunjing_knowledge_document`，并把导入项状态更新为 `IMPORTED`、`targetType=KNOWLEDGE`、`targetId=知识文档ID`。审核拒绝时不进入知识库。

采集来源运行接口用于承接外部爬虫、图影中华图片库采集器或人工整理后的发现项。接口只生成待审核资料，不直接公开发布：

```json
{
  "sourceId": 1,
  "items": [
    {
      "itemType": "WEB_PAGE",
      "itemTitle": "喀什古城图文资料页",
      "originalUrl": "https://www.example.com/kashgar/source",
```

## 5. 前端联调约定

- 一期小程序扫码后优先拿 `packageCode`，调用资源包详情接口。
- 正式扫码入口优先传 `sceneCode`，后台解析出 `packageCode` 和目标页；前端不需要硬编码资源包映射。
- `resourceType` 当前支持 `BOOK`、`MAP`、`GLOBE`。
- `mediaAssets` 承接图影中华等图片库素材，App 公开接口只返回 `reviewStatus=APPROVED`、`copyrightStatus=AUTHORIZED` 且 `canPublic=true` 的数据。
- 外部爬虫或图影中华采集器必须先调用后台采集运行接口生成待审核项，不能绕过审核直接写公开知识库或素材库。
- `knowledgeDocuments` 公开接口只返回 `reviewStatus=APPROVED` 且 `vectorStatus=INDEXED` 的数据。
- AI 生成和宣传物料选图必须分别检查 `canAiUse=true`、`canPromotionUse=true`，不能只看素材是否公开。
- P0 问答优先使用资源包 `aiKnowledgeId` 绑定的 Yudao AI 知识库段落召回；未绑定时使用本模块已审核知识文档兜底。
- 当前 answer 会在召回 sources 后调用 Yudao AI 默认 Chat 模型生成；模型不可用、异常或返回空内容时，降级为确定性 P0 讲解文本。
- `ai-quota-rules.cacheEnabled=true` 时，相同上下文的同日重复问题会命中 `xunjing_ai_generation_log` 缓存，返回原 sources，并写入 `cacheHit=true` 的新日志。
- AI 正式回答必须记录 `projectId`、`schoolId`、`packageId`、`qrCodeId`、`sourceJson`、`tokenCount`、`costAmount`、`safetyStatus`，并受 `ai-quota-rules` 的项目、学校、资源包、二维码、用户五维配额控制。
- 后台 AI 评测运行使用同一 App 问答链路，评测报告必须能定位到每题 `logId` 和 sources 数量。
- 所有真实资源 URL 必须来自对象存储或可信 CDN，不写死到小程序包里。
