# 星河寻境官网部署上线说明

更新时间：2026-06-21

本文档仅适用于星河寻境 / 图秀中华对外宣传官网静态页面部署，不适用于 Yudao 后台、小程序 API、AI 服务、知识库、Qdrant、对象存储等业务平台部署。业务平台部署见 `docs/02_开发规划/星河寻境业务平台部署说明.md`。

## 当前上线形态

当前版本是静态 React 官网，适合部署到 Nginx、宝塔静态站点、对象存储静态网站、CDN 源站或后续 Gitee Pages。预约演示表单支持运行时配置 HTTPS 线索 webhook；未配置时会写入浏览器本地兜底队列，不适合作为长期运营线索通道。

## 本地验证

```bash
npm install
npm run release:check
npm run release:package
```

验证通过后，构建产物在：

```text
dist/
tmp/releases/
```

```bash
npm run dev -- --port 5173
```

打开：

```text
http://localhost:5173/
```

建议检查：

- 桌面 1440x900 首屏。
- 移动 390x844 首屏。
- 顶部导航切换。
- 移动端菜单打开/关闭。
- 预约演示表单提交成功态。
- 浏览器控制台无 error/warn。

## Nginx 静态部署

建议使用版本化发布目录：

```text
/www/wwwroot/xinghexunjing/
  releases/
    20260621-010000/
      index.html
      assets/
      favicon.svg
  current -> releases/20260621-010000
```

Nginx 站点配置：

```nginx
server {
    listen 80;
    server_name your-domain.example;

    root /www/wwwroot/xinghexunjing/current;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location = /runtime-config.js {
        add_header Cache-Control "no-store";
        try_files $uri =404;
    }

    location ^~ /assets/ {
        expires 30d;
        add_header Cache-Control "public, immutable";
        try_files $uri =404;
    }
}
```

发布步骤：

```bash
npm ci
npm run release:package
```

上传 `tmp/releases/xinghexunjing-web-*.tar.gz` 到服务器，解压后把包内 `site/` 内容发布到新的 release 目录，并按正式线索接口改写 `runtime-config.js`：

```js
window.XINGHE_SITE_CONFIG = {
  leadWebhookUrl: 'https://your-crm.example.com/api/xinghe/leads'
};
```

然后切换软链：

```bash
ln -sfn /www/wwwroot/xinghexunjing/releases/YYYYMMDD-HHMMSS /www/wwwroot/xinghexunjing/current
nginx -t
nginx -s reload
```

## 上线后检查

- `https://your-domain.example/` 返回 200。
- 首页、`#capabilities`、`#scenarios`、`#solutions`、`#pilots`、`#travelogue`、`#about` 可打开。
- CSS、JS、图片资源无 404。
- 移动端无横向滚动。
- `runtime-config.js` 响应头包含 `Cache-Control: no-store`。
- 预约表单提交到正式 HTTPS 线索 webhook。

可用本仓库命令做部署后自动验证：

```bash
npm run deploy:verify -- https://your-domain.example/
```
