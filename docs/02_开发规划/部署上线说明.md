# 星河寻境官网部署上线说明

更新时间：2026-06-21

本文档仅适用于星河寻境 / 图秀中华对外宣传官网静态页面部署，不适用于 Yudao 后台、小程序 API、AI 服务、知识库、Qdrant、对象存储等业务平台部署。业务平台部署见 `docs/02_开发规划/星河寻境业务平台部署说明.md`。

## 当前上线形态

当前版本是 React 官网，适合部署到 Docker Compose、Nginx、宝塔静态站点、对象存储静态网站、CDN 源站或后续 Gitee Pages。Docker Compose 部署默认带本地线索采集服务，预约演示表单会提交到同域 `/api/leads` 并持久化为 JSONL；纯静态部署仍建议配置 HTTPS 线索 webhook，未配置时才会写入浏览器本地兜底队列。

## 本地验证

```bash
npm install
npm run release:check
npm run release:package
```

验证通过后，构建产物在：

```text

```

## 构建路径

默认构建适用于独立域名或站点根目录，例如：

```bash
npm run release:package
```

如果部署到 Gitee Pages 或 CDN 子路径，例如 `https://xinghetech.gitee.io/xinghexunjing/`，构建时必须设置：

```bash
VITE_BASE_PATH=/xinghexunjing/ npm run release:package
```

该配置会把首页里的 JS、CSS、`runtime-config.js`、`favicon.svg` 和 `site.webmanifest` 都指向 `/xinghexunjing/` 前缀，避免子路径部署后静态资源 404。独立域名部署不要设置该变量。

如果要准备 Gitee Pages 可直接发布的目录，使用：

```bash
VITE_BASE_PATH=/xinghexunjing/ \
XINGHE_LEAD_WEBHOOK_URL="https://your-crm.example.com/api/xinghe/leads" \
node scripts/prepare-gitee-pages.mjs
```

输出目录：

```text
tmp/gitee-pages/
```

该目录会包含 `index.html`、`404.html`、`.spa`、`.nojekyll`、`runtime-config.js` 和构建后的静态资源。

如果需要把该目录推送到 Gitee Pages 源分支，使用：

```bash
node scripts/publish-gitee-pages.mjs
```

默认会把 `tmp/gitee-pages/` 内容强制发布到远端 `gh-pages` 分支。也可以通过 `GITEE_PAGES_BRANCH`、`GITEE_PAGES_REMOTE_URL` 和 `GITEE_PAGES_COMMIT_MESSAGE` 覆盖目标分支、远端和提交信息。Gitee Pages 服务本身仍需要在 Gitee 项目后台启用或更新，并把源分支配置为实际发布分支。

发布分支推送后，先运行公网发布门禁：

```bash
node scripts/verify-gitee-pages-publication.mjs
```

该门禁会分别检查 `gh-pages` 分支、Gitee 仓库公开访问状态和 `https://xinghetech.gitee.io/xinghexunjing/` 页面状态。如果仓库公开访问返回 401、403 或 404，需要先把 Gitee 仓库开放为公网可访问，或使用支持私有仓库 Pages 的 Gitee 服务；如果页面仍返回 404，需要在 Gitee 项目后台启用或更新 Gitee Pages，并把源分支配置为 `gh-pages`。

正式把 Gitee Pages 当作可对外访问地址前，再运行最终 go-live 门禁：

```bash
node scripts/verify-gitee-go-live.mjs
```

该门禁会把 `master-sync`、`gh-pages` 产物提交号、仓库公网可见性和 Pages URL 内容检查合在一起。只有本地 `HEAD` 已推到 `origin/master`，并且 `gh-pages` 产物来自同一提交、公网仓库与 Pages URL 都可访问时，才可视为 Gitee Pages 上线通过。

## Nginx 静态部署

如果使用 Docker 直接部署，优先采用：

```bash
export XINGHE_WEB_PORT=8080
docker compose -f ops/compose.prod.yml up -d --build --wait
```

`ops/compose.prod.yml` 已包含自动重启、端口映射、网站健康检查、本地线索采集服务、`/api/leads` 代理和持久化卷。默认线索入口是同域 `/api/leads`；如果要改为 CRM、飞书、企业微信或后端 API，可额外设置：

```bash
export XINGHE_LEAD_WEBHOOK_URL="https://your-crm.example.com/api/xinghe/leads"
```

如果使用传统 Nginx 静态目录部署，优先使用发布包里的版本化部署脚本。

建议使用版本化发布目录：

```text
/www/wwwroot/xinghexunjing/
  releases/
    20260621-010000/
      index.html
      assets/
      favicon.svg
  current -> releases/20260621-010000
```

发布步骤：

```bash
npm ci
npm run release:package
```

上传 `tmp/releases/xinghexunjing-web-*.tar.gz` 到服务器后，解压发布包里的 `ops/` 目录，执行：

```bash
export SITE_ROOT=/www/wwwroot/xinghexunjing
export XINGHE_LEAD_WEBHOOK_URL="https://your-crm.example.com/api/xinghe/leads"
./ops/deploy-static-release.sh /tmp/xinghexunjing-web-YYYYMMDD-HHMMSS-<commit>.tar.gz
```

脚本会发布到新的 `releases/<release-id>/`，改写 `runtime-config.js`，切换 `current` 软链，并在 `nginx -t` 通过后执行 `nginx -s reload`。如服务器由宝塔、CI 或运维平台统一 reload，可临时加 `SKIP_NGINX_RELOAD=1`，但切换后仍必须手工执行 `nginx -t` 和 reload。

HTTPS 站点配置可从发布包内 `ops/nginx.https.conf.template` 复制，替换：

- `{{DOMAIN}}`
- `{{SITE_ROOT}}`
- `{{SSL_CERTIFICATE}}`
- `{{SSL_CERTIFICATE_KEY}}`

模板已包含 HTTP 跳 HTTPS、ACME challenge、SPA fallback、HSTS、安全响应头、`runtime-config.js` 禁止缓存和 `/assets/` 长缓存。

## 上线后检查

- `https://your-domain.example/` 返回 200。
- 首页、`#capabilities`、`#scenarios`、`#solutions`、`#pilots`、`#travelogue`、`#about` 可打开。
- CSS、JS、图片资源无 404。
- 移动端无横向滚动。
- `runtime-config.js` 响应头包含 `Cache-Control: no-store`。
- 预约表单提交到正式 HTTPS 线索 webhook，或 Docker Compose 默认的同域 `/api/leads`。
- 如使用 Compose 默认线索采集，`https://your-domain.example/api/leads/healthz` 返回 `xinghexunjing-leads ok`。

可用本仓库命令做部署后自动验证：

```bash
npm run deploy:verify -- https://your-domain.example/
npm run deploy:verify -- https://xinghetech.gitee.io/xinghexunjing/
```

正式对外运营前再运行公网 go-live 门禁：

```bash
node scripts/verify-go-live.mjs https://your-domain.example/
```

`deploy:verify` 可用于本地 HTTP、容器和预发静态站；`verify-go-live.mjs` 只用于正式公网域名，会强制检查 HTTPS、HSTS，以及真实 HTTPS 线索 webhook 或健康的同域 `/api/leads` 线索采集服务。
