## 正式源码结构

正式开发按下面结构推进：

```text
apps/
  mobile/      # 小程序 / H5 正式前端，由前端 AI 负责
  web/         # 星河寻境官网 / 图秀中华专题页，只做宣传和获客
backend/
  yudao/       # 独立复制出来的 Yudao 后台
    yudao-module-xunjing/ # 星河寻境业务、知识库、图片库、采集、AI 编排
packages/
  shared/      # API 类型、DTO、常量、Prompt 契约
data/
  seeds/       # 可选的大体量样本源文件；当前 MySQL 种子脚本以 backend/yudao/sql/mysql 为准
docs/
  02_开发规划/
```

不要再新增 `services/api` 作为主后端。后台能力必须进入 `backend/yudao`，优先通过 `yudao-module-xunjing` 承接。

喀什 P0 可落库样例脚本：

```text
backend/yudao/sql/mysql/xunjing-seed-kashgar-p0.sql
```

## 历史一期开发目标

以下目标来自 2026-06-21 的“图秀中华新疆首站”后台一期规划，作为已有代码和文档背景保留，不再作为当前西城 P0 默认范围：

1. 独立 Yudao 后台跑通。
2. `yudao-module-xunjing` 建立。
3. 一本书、一张地图、一个地球仪的扫码绑定。
4. 权威内容知识库最小闭环。
5. 图书扫码伴读、地图扫码讲解、RAG 问答带来源。
6. 基础素材库，记录来源、版权、审核和调用日志。
7. 扫码、问答、播放、资源包启用的基础看板。
8. 公益报告样例。

当前 `product/city-companion-main` 的西城 P0 已把 AI 旅行记录、游记生成、分享海报、PDF 纪念册、运营报告和多地区内容包复用纳入产品化规划，具体以 `docs/superpowers/specs/2026-06-27-xicheng-p0-full-loop-design.md` 为准。

## Git 提交建议

建议提交：

- `README.md`
- `docs/`
- `tools/`
- 后续新增的 `apps/`、`backend/`、`packages/`、`data/`
- 开发必需的小体积品牌素材

默认不提交：

- `archive/`
- `.DS_Store`
- `~$*` Office 锁文件
- 大型 GIF、MP4、zip
- PPT 生成过程中的 `backup/` 和 `tmp_review/`
- `.env`、`.env.local`、`.runtime/`、真实 API Key、Token、Cookie

推送长期分支和 release 分支时必须同步推送到 GitHub 和 Gitee。功能分支至少推 GitHub；需要国内设备接手时同步推 Gitee。
