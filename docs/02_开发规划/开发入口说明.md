## 正式源码结构

正式开发按下面结构推进：

```text
apps/
  mobile/      # 小程序 / H5 正式前端，由前端 AI 负责
  web/         # 星河寻境官网 / 图秀中华专题页，只做宣传和获客
backend/
  yudao/       # 独立复制出来的 Yudao 后台
    yudao-module-xunjing/ # 星河寻境业务、知识库、图片库、采集、AI 编排
packages/
  shared/      # API 类型、DTO、常量、Prompt 契约
data/
  seeds/       # 可选的大体量样本源文件；当前 MySQL 种子脚本以 backend/yudao/sql/mysql 为准
docs/
  02_开发规划/
```

不要再新增 `services/api` 作为主后端。后台能力必须进入 `backend/yudao`，优先通过 `yudao-module-xunjing` 承接。

喀什 P0 可落库样例脚本：

```text
backend/yudao/sql/mysql/xunjing-seed-kashgar-p0.sql
```

## 第一阶段开发目标

第一阶段聚焦“图秀中华新疆首站”的 P0 闭环，不做完整文旅平台：

1. 独立 Yudao 后台跑通。
2. `yudao-module-xunjing` 建立。
3. 一本书、一张地图、一个地球仪的扫码绑定。
4. 权威内容知识库最小闭环。
5. 图书扫码伴读、地图扫码讲解、RAG 问答带来源。
6. 基础素材库，记录来源、版权、审核和调用日志。
7. 扫码、问答、播放、资源包启用的基础看板。
8. 公益报告样例。

AI 旅行记录、游记生成、PDF 纪念页、复杂图片 AI、完整通用爬虫和多地区平台化能力全部后置。
