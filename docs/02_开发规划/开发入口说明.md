## 建议源码结构

正式开发时建议在 `01_星河寻境/` 内初始化仓库，并按下面结构新增源码：

```text
apps/
  web/        # 面向游客、文旅内容展示和互动体验
  admin/      # 内容、线路、素材、活动和发布管理后台
services/
  api/        # 后端 API、AI 编排、内容生成、素材管理
packages/
  shared/     # 共享类型、常量、工具函数
  ui/         # 可复用 UI 组件
```

## 建议第一阶段开发目标

第一阶段不建议直接做完整平台，先做一个可演示闭环：

1. PC Web 首页和移动端首页。
2. 文旅内容详情页。
3. AI 生成/推荐体验的前端假数据闭环。
4. 后台内容管理的最小原型。
5. 可从现有 PPT 和效果图反推的视觉规范。

## Git 提交建议

建议提交：

- `README.md`
- `docs/`
- `tools/`
- 后续新增的 `apps/`、`services/`、`packages/`
- 开发必需的小体积品牌素材

默认不提交：

- `archive/`
- `.DS_Store`
- `~$*` Office 锁文件
- 大型 GIF、MP4、zip
- PPT 生成过程中的 `backup/` 和 `tmp_review/`

推送代码时需要同步推送到 GitHub 和 Gitee。
