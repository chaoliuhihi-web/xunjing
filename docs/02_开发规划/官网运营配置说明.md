# 星河寻境官网运营配置说明

更新时间：2026-06-21

## 目标

官网第一阶段已经可以作为静态站点部署。为了支持真实对外运营，预约演示表单支持运行时配置线索 webhook，不需要重新打包前端。

## 线索 webhook

部署产物里包含：

```text
runtime-config.js
```

默认内容：

```js
window.XINGHE_SITE_CONFIG = {
  leadWebhookUrl: ''
};
```

上线时把 `leadWebhookUrl` 改为 CRM、企业微信机器人、飞书机器人、后端 API 或自动化平台的 HTTPS 地址：

```js
window.XINGHE_SITE_CONFIG = {
  leadWebhookUrl: 'https://your-crm.example.com/api/xinghe/leads'
};
```

表单会向该地址发送 `POST application/json`，字段包括：

- `name`
- `phone`
- `company`
- `type`
- `message`
- `source`
- `page`
- `url`
- `userAgent`
- `submittedAt`
- `status`

如果未配置 webhook，线索会写入浏览器 `localStorage` 的 `xinghe_xunjing_leads`，状态为 `pending-sync`。这只能作为兜底，不适合作为长期运营方案。

## Nginx 缓存要求

`runtime-config.js` 必须禁止缓存，避免运营人员改 webhook 后浏览器继续使用旧配置：

```nginx
location = /runtime-config.js {
    add_header Cache-Control "no-store";
    try_files $uri =404;
}
```

静态资源可以长期缓存：

```nginx
location ^~ /assets/ {
    expires 30d;
    add_header Cache-Control "public, immutable";
    try_files $uri =404;
}
```

## Docker 部署

本仓库包含 `Dockerfile` 和 `ops/nginx.conf`：

```bash
docker build -t xinghexunjing-web:latest .
docker run --rm -p 8080:80 xinghexunjing-web:latest
```

本地打开：

```text
http://localhost:8080/
```

## 发布前检查

```bash
npm ci
npm run release:check
npm run dev -- --port 5173
npm run qa:smoke
```

`qa:smoke` 会验证：

- 页面标题和首页核心文案。
- 桌面端无横向溢出。
- 移动端无横向溢出。
- 顶部导航切换到产品能力页。
- 跟着游记页可打开。
- 预约演示表单可提交并出现成功态。
- 浏览器控制台无 error/warn。
