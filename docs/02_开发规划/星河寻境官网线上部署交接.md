# 星河寻境官网线上部署交接

本文档用于把星河寻境官网部署到 `101.200.137.138`，并绑定域名 `xunjing.xingheai.net`。当前仓库以 Gitee 为可用远端，部署前必须先同步最新 `master`，重新打包，不要复用旧发布包。

## 1. 仓库与当前结论

- Gitee SSH：`git@gitee.com:xinghetech/xinghexunjing.git`
- Gitee Web：`https://gitee.com/xinghetech/xinghexunjing`
- GitHub 远端当前返回 `Repository not found`，不能作为交接来源。
- 服务器 IP：`101.200.137.138`
- 登录用户：`root`
- 登录方式：用户提供 PEM 私钥。
- 目标域名：`xunjing.xingheai.net`
- 目标站点目录：`/www/wwwroot/xinghexunjing`

当前探测结果：

- `80` / `443` 可访问，当前 Nginx 默认会跳到 `/aivideo_product_console/`。
- `8081` 是 HTTP JSON 服务，返回 `{"status":"ok","version":"1.0.3"}`，不是 SSH。
- `22` 端口超时，全端口扫描未发现其它 SSH 端口。
- `xunjing.xingheai.net` 当前未解析；`xingheai.net` 和 `www.xingheai.net` 已指向 `101.200.137.138`。
- 线上 HTTPS 证书当前只覆盖 `aivideo.xinghetech.cn`，需要为 `xunjing.xingheai.net` 单独签发证书。

## 2. 部署前必须解决

先在云服务器安全组和系统防火墙放行 SSH：

```bash
TCP 22
授权对象：执行部署机器的公网出口 IP
```

如果服务器实际使用非 22 SSH 端口，先向用户确认端口，再执行后续步骤。

本地验证 SSH：

```bash
chmod 600 /path/to/server.pem
ssh -i /path/to/server.pem -o StrictHostKeyChecking=accept-new root@101.200.137.138 'whoami; hostname; nginx -v'
```

## 3. 本地打包

```bash
git clone git@gitee.com:xinghetech/xinghexunjing.git
cd xinghexunjing
git checkout master
git pull --ff-only origin master
npm ci
npm run release:package
```

打包成功后会生成：

```text
tmp/releases/xinghexunjing-web-YYYYMMDD-HHMMSS-<commit>.tar.gz
tmp/releases/xinghexunjing-web-YYYYMMDD-HHMMSS-<commit>.manifest.json
```

`release:package` 内部会执行：

- `npm run test:run`
- `npm audit --audit-level=moderate`
- `npm run build`
- `node scripts/package-release.mjs`

## 4. 上传与版本化发布

```bash
scp -i /path/to/server.pem \
  tmp/releases/xinghexunjing-web-YYYYMMDD-HHMMSS-<commit>.tar.gz \
  root@101.200.137.138:/tmp/

ssh -i /path/to/server.pem root@101.200.137.138
mkdir -p /tmp/xinghexunjing-release
tar -xzf /tmp/xinghexunjing-web-YYYYMMDD-HHMMSS-<commit>.tar.gz -C /tmp/xinghexunjing-release
cd /tmp/xinghexunjing-release
SITE_ROOT=/www/wwwroot/xinghexunjing ./ops/deploy-static-release.sh /tmp/xinghexunjing-web-YYYYMMDD-HHMMSS-<commit>.tar.gz
```

发布脚本会创建：

```text
/www/wwwroot/xinghexunjing/
  releases/<release-id>/
  current -> releases/<release-id>
```

不要覆盖现有默认站点，不要改 `/aivideo_product_console/`。

## 5. Nginx 独立站点

优先使用发布包内模板：

```text
ops/nginx.https.conf.template
```

替换：

- `{{DOMAIN}}` -> `xunjing.xingheai.net`
- `{{SITE_ROOT}}` -> `/www/wwwroot/xinghexunjing`
- `{{SSL_CERTIFICATE}}` -> 新证书 fullchain 路径
- `{{SSL_CERTIFICATE_KEY}}` -> 新证书 private key 路径
- `{{YUDAO_APP_API_UPSTREAM}}` -> 暂设 `http://127.0.0.1:48080`

如果证书暂未签发，可以先创建 HTTP-only 独立 server 做预览：

```nginx
server {
    listen 80;
    server_name xunjing.xingheai.net;

    root /www/wwwroot/xinghexunjing/current;
    index index.html;

    location ^~ /assets/ {
        add_header Cache-Control "public, max-age=2592000, immutable" always;
        try_files $uri =404;
    }

    location = /runtime-config.js {
        add_header Cache-Control "no-store" always;
        try_files $uri =404;
    }

    location = /healthz.json {
        try_files $uri =404;
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

保存后必须执行：

```bash
nginx -t
nginx -s reload
```

## 6. 域名解析

在 `xingheai.net` DNS 控制台新增：

```text
记录类型：A
主机记录：xunjing
记录值：101.200.137.138
TTL：600 或默认
```

解析生效前可用本地强制解析验证：

```bash
curl -I --resolve xunjing.xingheai.net:80:101.200.137.138 http://xunjing.xingheai.net/
```

解析生效后签发 HTTPS 证书，再验证：

```bash
curl -I https://xunjing.xingheai.net/
npm run deploy:verify -- https://xunjing.xingheai.net/
node scripts/verify-go-live.mjs https://xunjing.xingheai.net/
```
