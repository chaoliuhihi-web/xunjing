# 西城 P0 后台上线状态

更新时间：2026-06-28
分支：`feature/xicheng-p0`

## 当前结论

状态：本地候选已通过 fresh local APP API readiness，NOT READY for 预发/生产。

本轮已把西城后台从“只有 Java 内置 5 个 POI 样例”推进到“有可初始化、可门禁校验、可 fresh local 启动联调的西城 POI/来源/评测 seed，并且 `/app-api/xunjing/triggers/resolve` 可优先读取 `xunjing_poi` 表”。触发识别侧已补 `TRIGGER_RESOLVE` 事件：APP 请求带 `packageCode` 时，后台会把识别结果和脱敏上下文写入 `xunjing_interaction_event`。APP 事件侧已补 `ERROR_FEEDBACK` 类型，支持西城 APP 用普通场景码上报识别失败、AI 反馈或素材异常。AI 对话侧已补 POI 来源约束：具体 POI 问答不能使用其他 POI 或泛化西城资料冒充来源，未命中匹配来源时返回 `BLOCKED`。本地依赖栈在缺少完整 Yudao baseline 时会导入 `xunjing-yudao-runtime-minimal.sql`，只用于 local-candidate App API readiness，不替代预发/生产的完整 Yudao baseline。新增的 `npm run xunjing:yudao:release:gate` 会把完整 Yudao baseline、HTTPS APP API 域名、微信 AppID、AI/OCR/视觉服务、对象存储、80 个以上西城已复核 POI 和来源授权作为生产 fail-closed 门禁。这还不是生产上线：P0 PRD 要求不少于 80 个已复核 POI，当前 seed 是 24 个本地候选点位，并且坐标和来源授权仍标记为 `REVIEW_REQUIRED`。


当前注意：

`packageCode` 是触发识别的包级边界。APP 已知当前产品包时必须传 `packageCode=XICHENG-MAP-001`；后台只会在该公开资源包内读取已发布且已审核的 `xunjing_poi` 候选，找不到包内匹配时返回确认/问答兜底，不得把同一区域其它资源包的 POI 拼进当前包的 `targetPath`。

- 触发引擎已优先读取 `xunjing_poi` 表；只有数据库为空或查询异常时才回落到 Java 内置 POI 样例。
- APP 若希望后台自动记录识别事件，必须传 `packageCode=XICHENG-MAP-001`。传入后响应会回显 `packageCode`，并在 `targetPath` 和候选 `targetPath` 中携带 `packageCode`，APP 可将识别结果的 `packageCode`、`regionCode`、`poiCode` 和 `poiName` 直接传给 `/app-api/xunjing/ai/chat`。不传 `packageCode` 时接口仍可识别，但后台不会自动归因并写入 `TRIGGER_RESOLVE` 事件。
- APP 不应上传 `photoMeta.imageBase64` 给事件链路；生产图片上传应走对象存储/上传凭证后只传图片标识或识别服务结果。

### AI 对话

接口：`POST /app-api/xunjing/ai/chat`

请求关键字段：

```json
{
  "packageCode": "XICHENG-MAP-001",
  "regionCode": "beijing-xicheng",
  "poiCode": "xicheng-baitasi",
  "poiName": "妙应寺白塔",
  "question": "白塔寺为什么重要？"
}
```

约束：

- 只允许基于 `review_status=APPROVED` 且 `vector_status=INDEXED` 的知识文档回答。
- 当请求带 `poiCode` 或 `poiName` 时，命中的来源必须匹配该 POI；其他 POI 或泛化西城资料不能作为成功回答来源。
- 无审核来源时返回 `BLOCKED`，不能编造。
- 涉及实时开放、票价、交通、活动时必须提示以官方实时渠道为准。

- 上传凭证与对象存储。
- 真实 AI Key。
- 生产/预发必须启用 Qdrant 向量检索和 Embedding：`SPRING_AI_VECTORSTORE_TYPE=qdrant`、`SPRING_AI_MODEL_EMBEDDING` 不得为 `none`、`DASHSCOPE_EMBEDDING_ENABLED=true`。
- P0 不少于 80 个已复核 POI；后续新增 POI 必须继续通过 `xicheng-poi-seed-quality`，确保每个点位都有别名、坐标、触发词、来源、摘要、推荐问题和审核状态。
- 真实 POI 数据包进入 production seed 前，可先运行 `npm run xunjing:xicheng:poi:review:pack` 生成带 24 个本地候选 POI 的草稿模板、逐点审核清单、可填写字段表和 review packet；模板不会通过 production manifest gate，checklist/workbook/review packet 也不能作为 evidence。需要显式传参时可使用底层模板命令：`npm run xunjing:xicheng:poi:manifest:template -- --output workbench/xicheng-production-pois.prefilled.json --seed-sql backend/yudao/sql/mysql/xunjing-seed-xicheng-p0.sql --review-checklist workbench/xicheng-production-pois.review-checklist.csv --review-workbook workbench/xicheng-production-pois.review-workbook.csv --review-packet workbench/xicheng-production-pois.review-packet.json`。审核人补齐真实资料后，先运行 `npm run xunjing:xicheng:poi:manifest:from-workbook -- --workbook workbench/xicheng-production-pois.review-workbook.csv --output workbench/xicheng-production-pois.json --production-ready --batch-code xicheng-p0-poi-review-YYYYMMDD --data-owner xicheng-cultural-tourism-review-team --source-compiled-by xicheng-source-compiler --source-compiled-at YYYY-MM-DD --reviewed-by xicheng-production-reviewer --reviewed-at YYYY-MM-DD --evidence-package-ref oss://xunjing-review/xicheng/review-batches/xicheng-p0-poi-review-YYYYMMDD.zip` 生成 manifest，再运行 `npm run xunjing:xicheng:poi:manifest:gate -- --manifest workbench/xicheng-production-pois.json --evidence-file qa/xicheng-poi-manifest-evidence.json`；只有输出 `PRODUCTION_POI_MANIFEST_READY` 且 checks 包含 `manifest-review-batch` 后，才能运行 `npm run xunjing:xicheng:poi:seed:generate -- --manifest workbench/xicheng-production-pois.json --output workbench/xicheng-poi-production-seed.sql --evidence-file qa/xicheng-poi-production-seed-generation-evidence.json` 生成 SQL 和生成证据；生成的 SQL 必须带 `xunjing_assert_xicheng_production_seed_ready` 前置检查，缺少 `XICHENG-MAP-001` 资源包或项目/学校引用时通过 `SIGNAL SQLSTATE '45000'` 中止。随后运行 `npm run xunjing:xicheng:poi:seed:verify -- --sql workbench/xicheng-poi-production-seed.sql --evidence-file qa/xicheng-poi-production-seed-evidence.json` 输出 `PRODUCTION_POI_SEED_READY` 后再人工审查和纳入迁移。

- 预发环境实机联调和 release evidence。
