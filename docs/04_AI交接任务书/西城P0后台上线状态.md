## 当前结论

状态：本地候选已通过 fresh local APP API readiness，NOT READY for 预发/生产。

本轮已把西城后台从“只有 Java 内置 5 个 POI 样例”推进到“有可初始化、可门禁校验、可 fresh local 启动联调的西城 POI/来源/评测 seed，并且 `/app-api/xunjing/triggers/resolve` 可优先读取 `xunjing_poi` 表”。触发识别侧已补 `TRIGGER_RESOLVE` 事件：APP 请求带 `packageCode` 时，后台会把识别结果和脱敏上下文写入 `xunjing_interaction_event`。APP 事件侧已补 `ERROR_FEEDBACK` 类型，支持西城 APP 用普通场景码上报识别失败、AI 反馈或素材异常。AI 对话侧已补 POI 来源约束：具体 POI 问答不能使用其他 POI 或泛化西城资料冒充来源，未命中匹配来源时返回 `BLOCKED`。本地依赖栈在缺少完整 Yudao baseline 时会导入 `xunjing-yudao-runtime-minimal.sql`，只用于 local-candidate App API readiness，不替代预发/生产的完整 Yudao baseline。新增的 `npm run xunjing:yudao:release:gate` 会把完整 Yudao baseline、HTTPS APP API 域名、微信 AppID、AI/OCR/视觉服务、对象存储、80 个以上西城已复核 POI 和来源授权作为生产 fail-closed 门禁。这还不是生产上线：P0 PRD 要求不少于 80 个已复核 POI，当前 seed 是 24 个本地候选点位，并且坐标和来源授权仍标记为 `REVIEW_REQUIRED`。

## APP AI 接口契约

### 识别地点

接口：`POST /app-api/xunjing/triggers/resolve`

请求关键字段：

```json
{
  "packageCode": "XICHENG-MAP-001",
  "regionCode": "beijing-xicheng",
  "sceneCode": "xicheng-multimodal-trigger",
  "sourceChannel": "xicheng-app",
  "userTraceId": "trace-xicheng-device-001",
  "text": "我在白塔寺附近",
  "ocrText": "妙应寺白塔",
  "location": {
    "latitude": 39.9231,
    "longitude": 116.35726,
    "coordType": "GCJ02"
  },
  "imageLabels": ["white_pagoda", "temple"],
  "recentPoiCodes": ["xicheng-baitasi"]
}
```

响应关键字段：

```json
{
  "regionCode": "beijing-xicheng",
  "poiCode": "xicheng-baitasi",
  "poiName": "妙应寺白塔",
  "confidence": 0.85,
  "requiresUserConfirm": false,
  "targetPath": "/pages/ai-guide/detail?regionCode=beijing-xicheng&poiCode=xicheng-baitasi",
  "suggestedQuestions": ["白塔寺为什么重要？"],
  "sources": [
    {
      "title": "妙应寺白塔",
      "sourceType": "OFFICIAL_PUBLIC",
      "sourceUrl": "https://www.bjxch.gov.cn/xxgk/zdly/jgxx/lyscjg/"
    }
  ]
}
```

当前注意：

- 触发引擎已优先读取 `xunjing_poi` 表；只有数据库为空或查询异常时才回落到 Java 内置 POI 样例。
- APP 若希望后台自动记录识别事件，必须传 `packageCode=XICHENG-MAP-001`。不传 `packageCode` 时接口仍可识别，但后台不会自动归因并写入 `TRIGGER_RESOLVE` 事件。
- APP 不应上传 `photoMeta.imageBase64` 给事件链路；生产图片上传应走对象存储/上传凭证后只传图片标识或识别服务结果。

### AI 对话

接口：`POST /app-api/xunjing/ai/chat`

请求关键字段：

```json
{
  "packageCode": "XICHENG-MAP-001",
  "regionCode": "beijing-xicheng",
  "poiCode": "xicheng-baitasi",
  "poiName": "妙应寺白塔",
  "question": "白塔寺为什么重要？"
}
```

约束：

- 只允许基于 `review_status=APPROVED` 且 `vector_status=INDEXED` 的知识文档回答。
- 当请求带 `poiCode` 或 `poiName` 时，命中的来源必须匹配该 POI；其他 POI 或泛化西城资料不能作为成功回答来源。
- 无审核来源时返回 `BLOCKED`，不能编造。
- 涉及实时开放、票价、交通、活动时必须提示以官方实时渠道为准。

### 错误反馈

接口：`POST /app-api/xunjing/resource/events`

请求关键字段：

```json
{
  "packageCode": "XICHENG-MAP-001",
  "sceneCode": "xicheng-ai-guide",
  "eventType": "ERROR_FEEDBACK",
  "sourceChannel": "xicheng-app",
  "userTraceId": "trace-xicheng-device-001",
  "payloadJson": "{\"category\":\"ocr_no_match\",\"message\":\"无法识别当前位置\",\"poiCode\":\"xicheng-unknown\",\"severity\":\"WARN\"}"
}
```

当前注意：带 `packageCode` 时，`sceneCode` 可以是普通 APP 场景码；如果不传 `packageCode`，`sceneCode` 仍必须能解析到已启用二维码。

## 剩余上线阻塞

- 真实微信 AppID。
- HTTPS 后端域名。
- OCR / 视觉识别服务和生产 API Key。
- 上传凭证与对象存储。
- 真实 AI Key。
- P0 不少于 80 个已复核 POI；后续新增 POI 必须继续通过 `xicheng-poi-seed-quality`，确保每个点位都有别名、坐标、触发词、来源、摘要、推荐问题和审核状态。
- 真实 POI 数据包进入 production seed 前，必须先运行 `npm run xunjing:xicheng:poi:manifest:gate -- --manifest workbench/xicheng-production-pois.json --evidence-file qa/xicheng-poi-manifest-evidence.json`，输出 `PRODUCTION_POI_MANIFEST_READY` 后，再运行 `npm run xunjing:xicheng:poi:seed:generate -- --manifest workbench/xicheng-production-pois.json --output workbench/xicheng-poi-production-seed.sql` 生成 SQL，并运行 `npm run xunjing:xicheng:poi:seed:verify -- --sql workbench/xicheng-poi-production-seed.sql --evidence-file qa/xicheng-poi-production-seed-evidence.json` 输出 `PRODUCTION_POI_SEED_READY` 后再人工审查和纳入迁移。
- 每个 P0 POI 的坐标、来源授权、内容审核、来源授权证据引用、现场测试照片证据引用和 `triggerSmokeStatus=PASSED`；来源授权证据和现场证据只能使用 `https://`、`oss://`、`cos://` 或 `s3://` 这类非本地引用，不能提交本地文件路径或 base64 图片。
- 预发/生产环境必须验证 `/triggers/resolve` 实际读取数据库 seed，而不是只看本地单元测试。
- 预发环境实机联调和 release evidence。
- `npm run xunjing:yudao:release:gate -- --stage production --env-file /secure/path/production.env --poi-manifest-evidence qa/xicheng-poi-manifest-evidence.json --poi-seed-evidence qa/xicheng-poi-production-seed-evidence.json --evidence-file qa/xicheng-yudao-release-evidence.json` 必须输出 `PRODUCTION_READY_CANDIDATE`，当前仓库状态仍应输出 `NOT_READY`；`--evidence-file` 只能写入 `qa/`、`tmp/` 或 `workbench/`，证据包不得包含 env 值或密钥。
- 最终发布审核前必须运行 `npm run xunjing:xicheng:release:evidence:package -- --stage production --release-evidence qa/xicheng-yudao-release-evidence.json --poi-manifest-evidence qa/xicheng-poi-manifest-evidence.json --poi-seed-evidence qa/xicheng-poi-production-seed-evidence.json --app-readiness-evidence qa/xicheng-app-readiness-evidence.json --evidence-file qa/xicheng-release-evidence-package.json` 并输出 `XICHENG_RELEASE_EVIDENCE_PACKAGE_READY`；最终证据包会直接复算 POI manifest/seed 源文件哈希，不能只提交孤立 JSON 摘要。
