# 下一阶段开发任务书

更新时间：2026-06-20

## 接手目标

基于现有星河寻境与图秀中华资料，启动一个可演示的 AI 文旅出版平台原型。目标不是一次性做完整平台，而是先打通“内容展示、移动浏览、AI 推荐感知、后台管理雏形、品牌视觉一致性”的最小闭环。

## 接手前必须阅读

1. `README.md`
2. `docs/00_项目总览/资料索引.md`
3. `docs/01_产品规划/`
4. `docs/02_开发规划/星河寻境 AI 文旅出版平台v0.2`

## 第一阶段建议任务

1. 初始化代码仓库和基础工程。
2. 建立 Web 首页、移动端首页和内容详情页。
3. 把现有 PPT/效果图里的视觉方向沉淀为基础主题、组件和页面布局。
4. 做一个 AI 文旅推荐体验的前端闭环，先用本地 mock 数据。
5. 建立后台管理最小页面，用于维护景点、线路、图文内容和发布状态。
6. 补开发命令、环境变量样例和 README，确保下一位工程师可以直接运行。

## 重要边界

- 不要把 `deliverables/` 当源码目录。
- 不要拆散 `workbench/ppt-generation/`，它是可复现 PPT 的工作台。
- 不要把 `archive/` 里的临时文件重新引入开发。
- 大型媒体素材默认本地保留，只有被页面或演示明确依赖时才考虑纳入仓库。
- 代码推送时需要同步推送到 GitHub 和 Gitee。
