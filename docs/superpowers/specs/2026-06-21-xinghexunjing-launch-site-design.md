# 星河寻境对外运营官网设计说明

更新时间：2026-06-21

## 目标

基于 `assets/references/Web_V3` 的 6 张长页参考图，交付一个可以本地构建、可部署到静态服务器、可对外演示和获客的星河寻境官网第一阶段版本。

## 范围

- 首页：品牌定位、核心能力、行业痛点、产品矩阵、界面展示、数据能力、转化 CTA、页脚。
- 产品能力页：平台架构、能力矩阵、AI 内容生产流程、企业级保障。
- 应用场景页：多元场景、合作流程、预约/合作咨询表单、FAQ、公司介绍。
- 解决方案页：文旅局、景区、展馆/博物馆、出版社/研学四类方案。
- 试点样板页：可复制样板、成效数据、方法论、资产模块、客户评价。
- 跟着游记页：旅行素材上传到游记/纪念书/分享海报的产品体验说明。

## 设计取舍

- 采用浅色科技文旅风格：白底、淡蓝山水、深蓝标题、蓝紫渐变按钮、玻璃卡片和轻量动效。
- 不把参考图整张截图直接当页面使用，真实导航、按钮、表单和文本均由代码渲染。
- 参考图中的大场景和产品效果图作为视觉资产和背景来源，保证高还原同时保留可维护性。
- 第一阶段以静态官网运营闭环为目标，表单用前端成功态记录需求，不接入真实 CRM/API。

## 技术方案

- Vite + React + CSS，仓库根目录直接构建静态官网。
- 使用 Vitest + Testing Library 约束导航、核心文案、页面切换和预约表单成功态。
- `npm run build` 生成 `dist/`，可直接部署到 Nginx 静态站点；SPA fallback 使用 `try_files $uri $uri/ /index.html`。

## 验收标准

- `npm run test:run` 通过。
- `npm run build` 通过。
- 桌面和移动端截图无横向滚动、按钮不换行、核心文案不溢出。
- 顶部导航可以切换核心页面，移动端菜单可打开关闭。
- 预约表单可以完成提交并出现客户可理解的成功状态。
