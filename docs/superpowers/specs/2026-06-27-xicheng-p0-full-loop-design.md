# 西城 AI 旅伴 P0 完整闭环设计

## 背景

本设计把西城 AI 旅伴从“识别、问答、路线、游记”的基础试运营版本，扩展为可运营、可传播、可汇报的 P0 闭环。

P0 不再只覆盖现场体验，还要覆盖三类结果：

1. 游客能完成一次真实西城 Citywalk。
2. 运营能发起路线任务、审核作品并复盘数据。
3. 对外能生成分享海报、PDF 纪念册和城市运营报告。

## P0 总目标

核心链路：

```text
西城首页
-> 拍照 / OCR / 定位识别
-> 小京讲解 / 推荐路线
-> 开始记录
-> 轨迹、照片、识别、备注沉淀为旅行素材
-> 生成游记草稿
-> 生成分享海报 / PDF 纪念册
-> 进入作品审核和城市运营报告
```

运营增长链路：

```text
路线专题
-> 路线护照
-> 打卡任务
-> 徽章达成
-> 亲子研学任务
-> 分享海报
-> 用户作品审核
-> 热门路线、热门 POI、作品传播统计
```

灵感导入链路：

```text
粘贴攻略文字 / 上传攻略图片 / 输入地点清单
-> AI 提取地点
-> 匹配西城官方 POI
-> 用户确认
-> 生成可走路线
-> 可加入路线护照或 Citywalk 记录
```

## P0 范围分层

### P0-Core：现场体验必须打通

- 西城首页：小京、扫一扫、问问小京、路线推荐、开始记录、游记生成、最近识别。
- 多模态识别：拍照、OCR、定位和文本进入 `/app-api/xunjing/triggers/resolve`。
- 小京讲解：识别到 POI 后进入讲解或问答，必须绑定审核过的知识来源。
- 推荐路线：从当前 POI 推荐下一站或一条完整 Citywalk。
- 前台轨迹记录：用户主动开始、暂停、继续、结束，前端本地缓存，后端批量接收。
- 旅行素材盒：照片、OCR、识别结果、定位、轨迹点、用户备注进入同一素材时间线。
- 游记草稿：基于素材时间线生成可编辑草稿。

### P0-运营增长：轻量但必须可运营

- 路线护照：每条运营路线有任务列表、打卡点、完成度。
- 打卡徽章：按路线、主题或任务发放徽章，不做复杂等级体系。
- 亲子研学任务：每条路线可配置 3-5 个问答、小观察或拍照任务。
- 分享海报：基于路线、徽章、照片和一句 AI 摘要生成图片海报。
- 作品审核：用户生成的游记、海报和纪念册进入后台审核流，默认不公开。

### P0-内容生产和汇报：做可交付版本

- 一键抄作业：首版只支持粘贴文字、地点清单和上传攻略截图，不直接抓取小红书或公众号原文。
- POI 匹配：AI 只提取地点和偏好，最终必须匹配到星河寻境西城 POI 或由用户手动确认。
- PDF 纪念册：首版生成固定模板 PDF，包含封面、路线地图、照片时间线、游记正文、知识卡片和徽章页。
- 城市运营报告：首版生成固定模板报告，包含访问量、识别量、路线完成、热门 POI、作品数、分享数、误触发和内容优化建议。

## 不做或后置

- 不做全网链接爬取，不复制小红书、公众号正文和图片。
- 不做后台静默长期定位。
- 不做复杂社交关系、排行榜和积分商城。
- 不做完全自由的旅游电商行程、酒店、机票、账单。
- 不做专业 GIS 绘制工具，地图只服务文旅路线、打卡和报告。

## 前端页面接入

### 首页

新增 `mode=xicheng` 分支，保留喀什现有页面。

首页模块顺序：

1. 小京问候和当前区域。
2. 一键导入灵感。
3. 拍照 / 扫一扫 / 问问小京。
4. 今日推荐路线。
5. 开始记录 Citywalk。
6. 路线护照和徽章进度。
7. 最近识别和旅行素材盒入口。
8. 生成游记。

### 识别结果页

接收多模态触发结果，展示：

- POI 候选。
- 置信度和触发原因。
- 继续讲解。
- 加入旅行素材。
- 推荐下一站。
- 反馈误识别。

### 路线详情页

展示：

- 路线地图和点位顺序。
- 预计时长、距离、适合人群。
- 路线护照任务。
- 亲子研学任务。
- 开始记录。
- 导出为分享海报。

### 旅行素材盒

展示按时间排序的素材：

- 照片。
- OCR 文本。
- 识别 POI。
- 轨迹停留点。
- 用户语音或文字备注。
- 任务完成记录。

### 游记生成页

输入素材时间线，输出：

- 游记草稿。
- 朋友圈短文。
- 小红书风格笔记。
- 分享海报。
- PDF 纪念册。

### 数据对象

- `inspiration_import`：导入灵感，存原始输入摘要、提取地点、匹配结果和用户确认。
- `route_passport`：路线护照，绑定路线、任务、徽章和完成规则。
- `checkin_event`：打卡事件，绑定用户、路线、POI、照片和任务。
- `study_task`：亲子研学任务，支持问答、观察、拍照和选择题。
- `badge_award`：徽章达成记录。
- `share_poster`：分享海报生成记录。
- `memorial_pdf`：PDF 纪念册生成记录。
- `city_ops_report`：城市运营报告生成记录。
- `work_review`：作品审核状态。

## 验收标准

### 用户侧

- 用户能从西城首页进入拍照、OCR、问答和路线。
- 用户能主动开始一条 Citywalk 记录。
- 用户拍照后能生成照片事件并进入素材盒。
- 用户能生成一篇可编辑游记草稿。
- 用户能生成一张分享海报。
- 用户能生成一个固定模板 PDF 纪念册。

### 运营侧

- 运营能配置路线护照、任务和徽章。
- 运营能审核用户游记、海报和 PDF 纪念册。
- 运营能查看热门 POI、热门路线、任务完成、作品数和误触发。
- 运营能生成一份城市运营报告。

### 技术侧

- `/app-api/xunjing/**` 请求必须带 `tenant-id`。
- 多模态识别不能在前端保存真实模型密钥。
- 用户分享默认隐藏精确经纬度。
- 导入灵感不保存第三方平台原文作为公开内容。

9. 补后台审核和运营看板。
